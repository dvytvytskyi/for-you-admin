# Альтернативні способи доступу до сервера

Якщо SSH не працює, спробуйте ці варіанти:

## 1. Перевірка через веб-консоль провайдера

Якщо ви використовуєте VPS провайдера (Hetzner, DigitalOcean, AWS тощо), зазвичай є веб-консоль:
- Увійдіть в панель управління провайдера
- Знайдіть "Console" або "VNC" доступ
- Підключіться через браузер

## 2. Перевірка SSH ключів

Якщо використовуєте SSH ключі:

```bash
# Перевірте чи є ваш публічний ключ на сервері
ssh-copy-id root@135.181.201.185

# Або додайте ключ вручну через веб-консоль
```

## 3. Перевірка пароля

Пароль для root: `FNrtVkfCRwgW`

Якщо пароль не працює:
- Можливо його змінили
- Спробуйте скинути через панель провайдера

## 4. Перевірка firewall

Якщо сервер має firewall (UFW, iptables), перевірте:

```bash
# Через веб-консоль або якщо вдалося підключитися
ufw status
iptables -L -n
```

## 5. Перевірка SSH сервісу

```bash
# Через веб-консоль
systemctl status ssh
systemctl status sshd
```

## 6. Альтернативний порт SSH

Якщо стандартний порт 22 заблокований, можливо SSH налаштований на інший порт:

```bash
# Спробуйте інші поширені порти
ssh -p 2222 root@135.181.201.185
ssh -p 22022 root@135.181.201.185
```

## 7. Через інший користувач

Якщо root заблокований, спробуйте іншого користувача:

```bash
ssh admin@135.181.201.185
ssh ubuntu@135.181.201.185
ssh debian@135.181.201.185
```

## 8. Перевірка через провайдера

Зв'яжіться з підтримкою провайдера, якщо:
- Сервер не відповідає
- Не можете підключитися через веб-консоль
- Потрібно скинути пароль

## 9. Локальна діагностика

На вашому комп'ютері:

```bash
# Перевірка DNS
nslookup 135.181.201.185
dig 135.181.201.185

# Перевірка порту
telnet 135.181.201.185 22
# або
nc -zv 135.181.201.185 22

# Детальна інформація про SSH
ssh -v root@135.181.201.185
```

## 10. Через інший комп'ютер/мережу

Спробуйте підключитися з іншого місця, щоб виключити проблеми з вашою мережею.
